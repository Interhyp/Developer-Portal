So starten Sie

In wenigen Schritten zur Schnittstellenanbindung

Wir freuen uns, dass Sie sich für unsere Schnittstellen interessieren. Um Ihnen die Integration möglichst einfach zu machen, finden Sie auf dieser Seite relevante Informationen zu den Voraussetzungen und dem Ablauf der Integration. Bitte lesen Sie die folgenden Punkte aufmerksam durch. Sollten noch Fragen offenbleiben, finden Sie weiter unten unsere Kontaktdaten.

Welche Technologien nutzen wir

Unsere APIs basieren auf der REST-Technologie und werden mit OAuth2 abgesichert.

Sie können somit über die bekannten HTTP Methoden Daten einliefern und Daten abrufen.

In den API-Beschreibungen haben Sie die Möglichkeit die technischen Schnittstellenbeschreibung bspw. als Yaml Datei herunterzuladen und basierend darauf Ihren Client zu generieren bzw. zu entwickeln.

Für die Integration und den Betrieb der Schnittstellen sollten Sie mit diesen Technologien vertraut sein oder über entsprechende IT-Unterstützung verfügen.

Überblick über das Developer Portal

Mit dem Developer Portal wollen wir Ihnen einen Überblick über unsere aktuell verfügbaren Schnittstellen ermöglichen und Ihnen Hilfestellungen bei der Integration geben.

Sie finden alle derzeit über dieses Portal veröffentlichten APIs im Menüpunkt API. Hier können Sie die einzelnen APIs selektieren und sich die implementierten Endpunkte und deren technische Spezifikation ansehen. Sie finden für jede Schnittstelle und jeden Endpunkt eine kurze Beschreibung und eine Angabe zum möglichen UseCase, für den diese Schnittstelle gedacht ist, sowie ggf. Einschränkungen oder weitere Hinweise.

Zusätzlich zur Beschreibung können Sie sich auch eine YAML oder JSON-Datei für die jeweilige Version des Endpunkts herunterladen.

Jede Schnittstellenbeschreibung verfügt auch über einen Link zum entsprechenden GitHub Repository. Auch hier können Sie sich die gewünschten Dateien und weitere Hilfestellungen, wie z. B. die Postman Collections mit Beispielaufrufen der Schnittstelle herunterladen.

Voraussetzungen für die Integration unserer Schnittstellen

Geschäftsverhältnis zur Prohyp

Für die Nutzung der meisten Schnittstellen benötigen Sie einen Vermittler-Account in ehyp – unsere Baufinanzierungsplattform. Dieser beinhaltet neben Ihren Kontaktdaten auch diverse individuelle Einstellungen, auf die sich unsere Schnittstellen beziehen.
Die Erstellung eines Accounts ist ganz einfach. Bitte nutzen Sie auf unserer Website www.prohyp.de die Funktion „Jetzt Partner werden“. Nach Eingabe Ihrer Kontaktdaten werden wir Ihren Account anlegen und Ihnen den Kooperationsvertrag zukommen lassen.

Unsere Nutzungsbedingungen

Für die Nutzung unserer Schnittstellen gelten bestimmte Nutzungsbedingungen. Diese erhalten Sie nach Anlage Ihres Accounts.  Bitte lesen Sie diese aufmerksam durch. Sollten Sie mit den Bedingungen nicht einverstanden sein, ist die Integration und Nutzung unserer Schnittstellen leider nicht möglich. Bei Fragen wenden Sie sich gerne an am@prohyp.de.

Authentifizierung über Token

Die Authentifizierung für unsere Schnittstellen erfolgt mittels Token. Diesen Token können Sie über die Authorization API  abrufen.  Die für den Aufruf dieser Schnittstelle relevanten Daten erhalten Sie nach Anlage Ihres Accounts.

Wir wollen Ihnen den Zugang zu unseren Schnittstellen möglichst einfach und einheitlich ermöglichen. Dennoch gibt es einige Schnittstellen, die aufgrund ihres Alters eine andere Methode der Authentifizierung benötigen. Die Information finden Sie in der jeweiligen Schnittstellenbeschreibung.

Benachrichtigung bei Änderungen

Wir arbeiten ständig an der Weiterentwicklung unserer Plattform, um allen Partnern das beste Nutzererlebnis bei der Vermittlung von Baufinanzierungen zu ermöglichen.

Dies beutet, dass sich bestehende Funktionen und Prozesse ändern können oder die Plattform um neue Features erweitert wird.

Diese Änderungen werden wir auch in unseren Schnittstellen abbilden. Um hier immer Up-to-date zu sein können Sie sich in GitHub Notifications für die jeweilige API abonnieren. Folgen Sie einfach dem Link nach GitHub, den Sie in der jeweiligen Schnittstellenbeschreibung finden und abonnieren („watchen“) sie das entsprechende Repository. Sie bekommen dann eine Benachrichtigung per Mail bei Anpassungen.

Grundsätzliches

Verfügbarkeit von veralteten Schnittstellen

Manchmal sind neue Features oder Technologien so andersartig, das Sie nicht in eine bestehende Schnittstelle integriert werden können. In diesem Fall werden wir eine neue Version der Schnittstelle bereitstellen. Um die neuen Features zu nutzen, müssen Sie auf die neue Version der Schnittstelle wechseln.

Wir werden die alte Version für einen gewissen Zeitraum weiter betreiben, um Ihnen die Möglichkeit zur Planung des technischen Umbaus zu geben.

Schnittstellen, die veraltet sind, werden in der Schnittstellenbeschreibung als DEPRECATED markiert und zu einem bestimmten Zeitpunkt deaktiviert. Wir empfehlen diese Schnittstellen nicht mehr neu zu implementieren, sondern die aktuelle Version zu nutzen.

Tolerant-Reader-Patterns

Bestehende Schnittstellen werden stetig weiterentwickelt. Somit können weitere Daten ausgeliefert oder neue Endpunkte hinzugefügt werden.

Unser Ziel ist es neue Funktionalitäten ohne "Breaking Changes" an der Schnittstelle hinzuzufügen.

Hierzu muss der Client das Tolerant-Reader-Pattern unterstützen, dies bedeutet unbekannte Felder dürfen nicht als Fehler angesehen werden, sondern müssen ignoriert werden.

Bei Fragen

Wenn im Zuge der Integration oder des Betriebs unserer Schnittstellen Fragen auftreten, die sich durch die Schnittstellenbeschreibung und die bereitgestellten Beispieldateien, (z.B. Postman-Collections für beispielhafte Schnittstellenaufrufe) nicht beantworten lassen, können Sie sich gerne an am@prohyp.de wenden.
